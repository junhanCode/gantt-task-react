# Demo 演示更新说明

## 📋 更新内容

在示例项目（example/src/App.tsx）中添加了时间轴格式自定义的演示面板，让用户可以直观地看到各种格式配置。

## 🎨 新增演示面板

### 时间轴格式演示面板

位置：渲染完成事件面板下方

#### 面板内容

1. **月模式 - 月份格式**
   - 当前格式：M7
   - 示例：M1, M2, M7, M12
   - 配置代码：`monthLabel: (m) => \`M${m+1}\``

2. **日/周模式 - 周标签格式**
   - 当前格式：Week 01
   - 示例：Week 01, Week 02, Week 52
   - 配置代码：`weekLabel: (w) => \`Week ${w.padStart(2,'0')}\``

3. **周模式 - 年月格式**
   - 当前格式：2026 11Mon
   - 示例：2026 01Mon, 2026 11Mon
   - 配置代码：`yearMonthLabel: (y,m) => \`${y} ${m+1}Mon\``

#### 其他可用格式提示

面板底部显示了其他可用的格式选项：
- 月份：Mon 7, 7月, 07
- 周：W01, 第1周, #01
- 年月：2026 11M, 2026-11, 2026/11

## 🚀 如何查看

1. 启动示例项目
   ```bash
   cd example
   npm start
   ```

2. 在浏览器中打开 `http://localhost:3000`

3. 查看演示面板
   - 在页面顶部可以看到时间轴格式演示面板
   - 切换不同视图模式（日/周/月）查看实际效果

4. 对比效果
   - 月模式：时间轴显示 M1, M2, M7
   - 周模式：母表头显示 2026 11Mon，子表头显示 Week 01
   - 日模式：母表头显示 Week 01，子表头显示日期

## 💡 使用指南

### 如何修改格式

1. 打开配置文件：`src/i18n/index.ts`

2. 找到对应语言的配置（`zhTW` 或 `en`）

3. 修改对应的函数：
   ```typescript
   const zhTW: I18nTexts = {
     // ...
     monthLabel: (m) => `M${m + 1}`,           // 月份格式
     weekLabel: (w) => `Week ${w.padStart(2, '0')}`,  // 周格式
     yearMonthLabel: (y, m) => `${y} ${m + 1}Mon`,    // 年月格式
   };
   ```

4. 重新编译查看效果
   ```bash
   npm run build
   cd example
   npm start
   ```
